## 训练前准备

### ✅ 检查清单

- [ ] Python 3.7+ 已安装
- [ ] 所有依赖包已安装（`pip install -r requirements.txt`）
- [ ] 预训练权重已下载（`03_model/pretrained/rgb_imagenet.pt`）
- [ ] 视频文件已放入 `01_data/raw_videos/`
- [ ] 标注文件已创建（至少30个样本/类别）
- [ ] 已完成数据预处理

### 1️⃣ 准备训练数据

#### 步骤1：准备视频
将羽毛球比赛视频放入：
```
01_data/raw_videos/
├── match_001.mp4
├── match_002.mp4
└── match_003.mp4
```

**视频要求**：
- 格式：MP4、AVI、MOV、MKV
- 分辨率：建议720p或以上
- 帧率：建议30fps
- 内容：清晰的羽毛球比赛视频

#### 步骤2：创建标注文件

在 `01_data/annotations/` 目录下为每个视频创建标注文件。

**文件命名**：`视频名_annotations.json`

**标注模板**：
```json
{
  "video_name": "match_001",
  "total_frames": 5400,
  "frame_rate": 30,
  "annotations": [
    {
      "action_id": 1,
      "class": "round_start",
      "start_frame": 100,
      "end_frame": 120,
      "description": "第1回合发球"
    },
    {
      "action_id": 2,
      "class": "round_end",
      "start_frame": 450,
      "end_frame": 470,
      "description": "第1回合球落地"
    }
  ]
}
```

**标注要求**：
- ✅ 每个类别至少30个样本
- ✅ 推荐50+个样本获得更好效果
- ✅ 标注准确，动作边界清晰
- ✅ 包含不同角度、光照条件的视频

**如何获取帧号**：
```python
import cv2

cap = cv2.VideoCapture('match_001.mp4')
total_frames = int(cap.get(cv2.CAP_PROP_FRAME_COUNT))
fps = cap.get(cv2.CAP_PROP_FPS)

print(f"总帧数: {total_frames}")
print(f"帧率: {fps} FPS")
print(f"时长: {total_frames/fps:.2f} 秒")
```

详细标注指南请查看：`06_docs/标注指南.md`

## 训练参数调整

### 📝 配置文件位置
`05_config/config.yaml`

### 🔧 关键参数说明

```yaml
training:
  # 基础参数
  batch_size: 2              # 批次大小（显存不足时改为1）
  epochs: 20                 # 训练轮数（建议15-30）
  learning_rate: 0.0001      # 学习率（建议0.0001-0.001）
  sequence_length: 32        # 输入帧序列长度（固定32）
  
  # 数据划分
  train_split: 0.8           # 训练集比例（80%训练，20%验证）
  
  # 优化器
  optimizer: "adam"          # 优化器类型
  weight_decay: 0.0001       # 权重衰减（L2正则化）
  
  # 学习率调度
  lr_scheduler:
    enabled: true            # 是否启用
    type: "step"             # 调度类型
    step_size: 5             # 每5轮降低学习率
    gamma: 0.5               # 学习率衰减系数
```

### 💡 参数调整建议

#### 显存不足
```yaml
training:
  batch_size: 1              # 从2改为1
  sequence_length: 16        # 从32改为16（可选）
```

#### 提高准确率
```yaml
training:
  epochs: 30                 # 增加训练轮数
  learning_rate: 0.00005     # 降低学习率
```

#### 加快训练
```yaml
training:
  epochs: 10                 # 减少训练轮数
  batch_size: 4              # 增加批次大小（需要更多显存）
```

## 训练过程监控

### 📊 关键指标

#### 1. 训练损失（Loss）
- **含义**：模型预测与真实标签的差距
- **期望**：逐渐下降
- **正常范围**：0.1-0.5

#### 2. 训练准确率（Accuracy）
- **含义**：训练集上的分类准确率
- **期望**：逐渐上升
- **目标**：>90%

#### 3. 验证损失
- **含义**：验证集上的损失
- **期望**：逐渐下降
- **注意**：如果上升可能过拟合

#### 4. 验证准确率
- **含义**：验证集上的准确率
- **期望**：逐渐上升
- **目标**：>85%

### 📈 训练曲线示例

```
Epoch  Train Loss  Train Acc  Val Loss  Val Acc
  1      0.6931      0.5000    0.7123    0.5000
  2      0.4567      0.7500    0.5234    0.6667
  3      0.3456      0.8750    0.4123    0.8333
  4      0.2345      0.9167    0.3567    0.8333
  5      0.1789      0.9583    0.3234    0.9167  ← 最佳
  ...
 20      0.0567      0.9792    0.3456    0.9000
```

### ⚠️ 异常情况

#### 过拟合（Overfitting）
**症状**：
- 训练准确率很高（>95%）
- 验证准确率较低（<80%）
- 验证损失上升

**解决方案**：
```yaml
training:
  weight_decay: 0.001        # 增加正则化
  epochs: 15                 # 减少训练轮数
```

#### 欠拟合（Underfitting）
**症状**：
- 训练准确率低（<80%）
- 验证准确率也低（<75%）

**解决方案**：
```yaml
training:
  epochs: 30                 # 增加训练轮数
  learning_rate: 0.0005      # 提高学习率
```

## 常见问题

### Q1: 显存不足（CUDA out of memory）

**症状**：
```
RuntimeError: CUDA out of memory. Tried to allocate 2.00 GiB
```

**解决方案**：
```yaml
# 修改 05_config/config.yaml
training:
  batch_size: 1              # 降低批次大小
```

或使用CPU训练：
```yaml
system:
  device: "cpu"              # 改为cpu
```

---

### Q2: 标注文件格式错误

**症状**：
```
✗ 标注文件格式错误: 缺少字段 'class'
```

**解决方案**：
1. 检查JSON格式是否正确
2. 确保包含必要字段：`class`, `start_frame`, `end_frame`
3. 运行验证：
```bash
cd 02_code
python data_preprocess.py
# 选择 [3] 验证标注文件
```

---

### Q3: 预训练权重未找到

**症状**：
```
FileNotFoundError: 预训练权重不存在: ../03_model/pretrained/rgb_imagenet.pt
```

**解决方案**：
1. 下载预训练权重：https://github.com/piergiaj/pytorch-i3d
2. 放置到：`03_model/pretrained/rgb_imagenet.pt`

---

### Q4: 训练准确率不高

**可能原因**：
- 数据量太少
- 标注质量不高
- 训练轮数不够

**解决方案**：
1. 增加训练数据（至少50个样本/类别）
2. 检查标注准确性
3. 增加训练轮数（30 epochs）
4. 调整学习率

---

### Q5: 训练速度很慢

**原因**：
- 使用CPU训练
- 批次大小太小

**解决方案**：
1. 使用GPU训练
2. 增加批次大小（如果显存允许）
```yaml
training:
  batch_size: 4              # 从2增加到4
```

---

### Q6: 如何恢复中断的训练

**方法1**：使用最后保存的模型继续训练
```python
# 修改 model_train.py
# 在加载预训练权重的地方改为：
model.load_state_dict(torch.load('path/to/last_model.pth'))
```

**方法2**：重新开始训练
- 训练会自动保存最佳模型
- 即使中断，最佳模型已保存
