## 二、数据流与处理流程

### 2.1 训练阶段数据流

```
原始视频 (MP4/AVI/MOV)
    ↓
VideoPreprocessor.extract_frames()  — OpenCV 逐帧读取
    ↓
ROI 裁剪 (可选, 950×720) → 尺寸调整 (224×224) → JPEG 保存
    ↓
标注文件 (JSON) 合并为 merged_labels.json
    ↓
BadmintonDataset._prepare_samples()  — 以标注中心帧截取 16 帧序列
    ↓
数据增强 (训练模式): 水平翻转、随机裁剪、色彩抖动、高斯模糊、随机旋转
    ↓
DataLoader → BadmintonI3D (InceptionI3d + 替换后的 logits 层)
    ↓
Adam 优化器 + CrossEntropyLoss → 保存 best_model/final_model
```

### 2.2 推理阶段数据流

```
原始视频上传
    ↓
ActionPredictorFast.predict_video()  — 流式分块读取
    ↓
滑动窗口提取帧序列 (sequence_length=16, stride=8)
    ↓
I3D 模型推理 + Softmax → 每窗口动作概率
    ↓
后处理: 置信度过滤(>0.1) → NMS 去重(min_interval=15帧) → 平滑滤波
    ↓
YOLO 辅助验证: 运动员检测、就位检测、羽毛球跟踪
    ↓
回合配对: round_start 与后续最近 round_end 配对
    ↓
时长约束过滤 (2–30秒) + 间隔合并 (<2秒) + 双人验证
    ↓
FFmpeg 精确剪辑 → 输出回合片段 MP4
```

## 三、核心技术详解

### 3.1 I3D（Inflated 3D ConvNet）

#### 3.1.1 网络结构

`InceptionI3d` 继承自 `nn.Module`，基于 Inception-v1 的 3D 扩展版本：

```python
输入张量: (Batch, 3, T, H, W)  — 默认 (B, 3, 16, 224, 224)

层结构:
  Conv3d_1a_7x7   (stride=2, 64 通道)
  MaxPool3d_2a_3x3 (stride=2)
  Conv3d_2b_1x1   (64 通道)
  Conv3d_2c_3x3   (192 通道)
  MaxPool3d_3a_3x3 (stride=2)
  Mixed_3b / Mixed_3c  — Inception 模块 (256 通道输出)
  MaxPool3d_4a_3x3 (stride=2)
  Mixed_4b ~ Mixed_4f  — 5 个 Inception 模块 (480→512→512→512→528 通道)
  MaxPool3d_5a_2x2 (stride=2)
  Mixed_5b / Mixed_5c  — 2 个 Inception 模块 (832→1024 通道)
  AdaptiveAvgPool3d    — 替代固定尺寸 AvgPool，支持任意输入分辨率
  Dropout(0.5)
  logits (Unit3D, 1×1×1, 输出通道=num_classes)
```

#### 3.1.2 InceptionModule 设计

```python
输入
  ├─ 1×1×1 Conv ─────────────────────────────→ b0
  ├─ 1×1×1 Conv → 3×3×3 Conv (padding=1) ────→ b1
  ├─ 1×1×1 Conv → 3×3×3 Conv (padding=1) ────→ b2
  └─ MaxPool3d → 1×1×1 Conv ─────────────────→ b3
                      ↓
              Concatenate (dim=1, 通道维度)
                      ↓
                    输出
```

四个分支分别提取不同尺度的时空特征，通过通道拼接实现多尺度融合。

#### 3.1.3 迁移学习实现

```python
class BadmintonI3D(nn.Module):
    def __init__(self, num_classes=2):
        super().__init__()
        self.model = InceptionI3d(num_classes=num_classes, in_channels=3)
        # 替换原始 logits 层适配二分类
        self.model.logits = Unit3D(
            in_channels=1024, output_channels=num_classes,
            kernel_shape=[1, 1, 1], activation_fn=None,
            use_batch_norm=False, use_bias=True, name='logits'
        )
        self.model.end_points['Logits'] = self.model.logits

    def forward(self, x):
        logits = self.model(x)
        # 处理不同输出维度
        if len(logits.shape) == 5:
            logits = logits.squeeze(-1).squeeze(-1)
        if len(logits.shape) == 3:
            logits = torch.mean(logits, dim=2)
        return logits
```

#### 3.2.2 ShuttlecockTracker — 两阶段回合结束检测

核心状态机实现：

```
状态: watching ──────────────────────────────────────────────────────────→ ball_down
         │                                                                        │
         │  触发条件:                                                             │
         │  ① 球连续消失 >= vanish_duration (默认 1.5s)                            │
         │  ② 或球位置标准差 < still_threshold 持续 >= still_duration (默认 0.5s)   │
         │                                                                        │
         │                                                                        ↓
         │                                                              等待球回手中
         │                                                                        │
         │                                                                        ↓
         │                                                            球出现在运动员框内
         │                                                                        │
         └────────────────────────────────────────────────────────────────────────┘
                                         触发 round_end，进入 3 秒冷却期
```

关键参数：
- `still_threshold=8.0` 像素：判定球静止的位置标准差阈值
- `vanish_duration=1.5` 秒：避免飞行中球暂时离开画面导致的误触发
- `cooldown_seconds=3.0`：防止同一回合多次触发

#### 3.2.3 SimplePersonTracker — 轻量人员跟踪

采用贪心最近邻匹配策略：

1. 计算所有检测框与已有轨迹末位置的欧氏距离
2. 按距离升序排序，依次分配 ID（距离 < max_distance=250 像素）
3. 未匹配的检测框创建新轨迹
4. 轨迹合并：间隔 <30 帧且边界距离 <250 像素的轨迹视为同一人

身份筛选策略：以 `green_court_count`（绿色场地内出现次数）排序，取前 2 名作为主运动员。

#### 3.3.1 算法流程

```python
def predict_video(video_path, sliding_stride=8, progress_callback=None):
    cap = cv2.VideoCapture(video_path)
    total_frames = int(cap.get(cv2.CAP_PROP_FRAME_COUNT))
    
    predictions = []
    for center in range(sequence_length // 2, total_frames, sliding_stride):
        start = center - sequence_length // 2
        end = center + sequence_length // 2
        frames = [read_and_preprocess(cap, i) for i in range(start, end)]
        
        input_tensor = torch.stack(frames).unsqueeze(0)  # (1, 3, 16, 224, 224)
        with torch.no_grad():
            if use_fp16 and device == cuda:
                input_tensor = input_tensor.half()
            logits = model(input_tensor)
            probs = F.softmax(logits, dim=1)
        
        predictions.append({
            'center_frame': center,
            'class': argmax(probs),
            'confidence': max(probs)
        })
    
    return post_process(predictions)
```

#### 3.3.2 后处理流水线

```
原始预测序列
    ↓
置信度过滤: 保留 confidence >= threshold (默认 0.1) 的预测
    ↓
NMS 去重: 按置信度降序排序，若两预测帧间隔 < min_interval (默认 15 帧) 则丢弃低置信度者
    ↓
类别分离: 拆分为 round_start 列表与 round_end 列表
    ↓
回合配对: 每个 round_start 匹配后续最近的 round_end，满足时长约束 (2–30 秒)
    ↓
间隔合并: 若相邻回合间隔 < merge_interval (默认 2 秒)，合并为单个回合
    ↓
YOLO 验证: 双人验证、就位检测、绿色场地比例过滤
    ↓
输出最终回合列表
```

## 五、算法原理

### 5.1 3D 卷积 vs 2D 卷积

**2D 卷积**（图像分类）：
- 输入: $(H, W, C)$
- 核: $(K_h, K_w, C)$
- 输出: $(H', W', C')$
- 局限: 无法建模时间维度动态信息

**3D 卷积**（视频动作识别）：
- 输入: $(T, H, W, C)$
- 核: $(K_t, K_h, K_w, C)$
- 输出: $(T', H', W', C')$
- 优势: 单一层即可同时捕捉空间外观与运动动态

计算式：
$$
\text{Output}[t, h, w, c] = \sum_{dt=0}^{K_t-1} \sum_{dh=0}^{K_h-1} \sum_{dw=0}^{K_w-1} \sum_{cin=0}^{C-1} \text{Input}[t+dt, h+dh, w+dw, cin] \times \text{Kernel}[dt, dh, dw, cin, c]
$$

### 5.2 迁移学习原理

```
Kinetics-400 预训练模型 (400 类通用动作)
           ↓
    加载 rgb_imagenet.pt 权重
           ↓
    替换 logits 层: 1024 → 2 (round_start / round_end)
           ↓
    训练策略:
      - 底层 Inception 模块: 小学习率微调 (lr × 0.1)
      - 顶层 logits 层: 正常学习率训练
      - BatchNorm momentum = 0.001 (小 batch 稳定统计)
           ↓
    羽毛球动作识别专用模型
```

### 5.3 损失函数与优化器

**交叉熵损失**：
$$
\mathcal{L} = -\sum_{i=0}^{N-1} \sum_{c=0}^{C-1} y_{i,c} \cdot \log(\hat{y}_{i,c})
$$

**Adam 优化器**：
$$
\begin{aligned}
m_t &= \beta_1 \cdot m_{t-1} + (1 - \beta_1) \cdot g_t \\
v_t &= \beta_2 \cdot v_{t-1} + (1 - \beta_2) \cdot g_t^2 \\
\theta_t &= \theta_{t-1} - \alpha \cdot \frac{m_t}{\sqrt{v_t} + \epsilon}
\end{aligned}
$$

默认参数: $\alpha=0.001$, $\beta_1=0.9$, $\beta_2=0.999$, $\epsilon=10^{-8}$

## 七、性能优化策略

### 7.1 训练优化

| 优化手段 | 实现方式 | 效果 |
|---------|---------|------|
| 数据增强 | 8 种组合（翻转、裁剪、抖动、模糊、旋转） | 有效扩充样本，降低过拟合 |
| 小 batch 稳定 | BatchNorm momentum=0.001 | 适配 batch_size=1 场景 |
| 预训练初始化 | Kinetics-400 + ImageNet 双预训练 | 加速收敛，提升泛化 |
| 自适应输入尺寸 | AdaptiveAvgPool3d 替代固定池化 | 支持 224×224 与原始分辨率 |

### 7.2 推理优化

| 优化手段 | 实现方式 | 效果 |
|---------|---------|------|
| 流式处理 | 分块读取视频帧，不全部驻留内存 | 支持任意长度视频，防 OOM |
| FP16 半精度 | `model.half()` + `input_tensor.half()` | GPU 推理加速 1.5–2 倍 |
| 自适应 batch | 根据显存容量自动调整 (4/8/16) | 最大化硬件利用率 |
| 增大滑动步长 | stride 从 8 提升至 16 | 窗口数减半，速度翻倍 |
| YOLO 降采样 | 运动员每 2 帧检测、羽毛球每 3 帧检测 | 减少冗余计算 |

### 7.3 视频处理优化

| 优化手段 | 实现方式 | 效果 |
|---------|---------|------|
| FFmpeg 快速剪辑 | `-c copy` 模式（Web 下载） | 秒级完成，无重编码 |
| 质量可控剪辑 | `-crf 23 -preset medium`（CLI 批量） | 平衡画质与文件大小 |
| 异步检测 | 后台线程 + 前端轮询 | UI 不卡顿，支持大视频 |
| HTTP 206 范围请求 | `Range: bytes=start-end` | 大视频支持拖动播放 |
