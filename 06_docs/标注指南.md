## 标注概述

### 1. 标注目的

为训练I3D动作识别模型提供高质量的标注数据，使模型能够准确识别羽毛球比赛中的关键动作。

### 2. 标注内容

本项目需要标注两类动作：

| 动作类别 | 英文标识 | 说明 |
|---------|---------|------|
| 发球 | `round_start` | 发球方球拍击球瞬间 |
| 球落地 | `round_end` | 球接触地面瞬间 |

### 3. 标注数量要求

| 类别 | 最少样本数 | 推荐样本数 |
|------|-----------|-----------|
| 发球 | 30 | 50+ |
| 球落地 | 30 | 50+ |
| 总计 | 60 | 100+ |

## 标注工具

### 1. 推荐工具

#### 1.1 VLC Media Player

**优点**：
- 免费开源
- 支持逐帧播放
- 显示当前帧号和时间戳

**使用方法**：
1. 打开视频：`媒体` → `打开文件`
2. 逐帧播放：按 `E` 键（下一帧）或 `Shift+左箭头`（上一帧）
3. 查看时间：`工具` → `媒体信息` → `编解码器`

#### 1.2 FFmpeg

**获取视频信息**：

```bash
# 查看视频总帧数和帧率
ffmpeg -i video.mp4

# 提取特定帧为图片
ffmpeg -i video.mp4 -vf "select=eq(n\,100)" -vframes 1 frame_100.jpg
```

#### 1.3 Python脚本

**快速查看帧**：

```python
import cv2

def show_frame(video_path, frame_number):
    cap = cv2.VideoCapture(video_path)
    cap.set(cv2.CAP_PROP_POS_FRAMES, frame_number)
    ret, frame = cap.read()
    
    if ret:
        cv2.imshow(f'Frame {frame_number}', frame)
        cv2.waitKey(0)
        cv2.destroyAllWindows()
    
    cap.release()

# 使用示例
show_frame('match_001.mp4', 100)
```

### 2. 获取帧号

#### 2.1 使用VLC

1. 打开视频
2. 暂停到目标位置
3. 查看时间戳（如 00:03.33）
4. 计算帧号：`帧号 = 时间戳(秒) × 帧率`
   - 例如：3.33秒 × 30fps = 100帧

#### 2.2 使用Python

```python
import cv2

def get_frame_info(video_path):
    cap = cv2.VideoCapture(video_path)
    
    total_frames = int(cap.get(cv2.CAP_PROP_FRAME_COUNT))
    fps = cap.get(cv2.CAP_PROP_FPS)
    duration = total_frames / fps
    
    print(f"总帧数: {total_frames}")
    print(f"帧率: {fps} FPS")
    print(f"时长: {duration:.2f} 秒")
    
    cap.release()

get_frame_info('match_001.mp4')
```

## 标注流程

### 1. 准备工作

#### 1.1 创建标注文件

为每个视频创建对应的JSON标注文件：

**文件命名规则**：`视频名_annotations.json`

**文件位置**：`01_data/annotations/`

**模板**：

```json
{
  "video_name": "match_001",
  "total_frames": 0,
  "frame_rate": 30,
  "annotations": []
}
```

#### 1.2 填写视频信息

```python
# 使用Python获取视频信息
import cv2
import json

video_path = '01_data/raw_videos/match_001.mp4'
cap = cv2.VideoCapture(video_path)

video_info = {
    "video_name": "match_001",
    "total_frames": int(cap.get(cv2.CAP_PROP_FRAME_COUNT)),
    "frame_rate": int(cap.get(cv2.CAP_PROP_FPS)),
    "annotations": []
}

cap.release()

# 保存模板
with open('01_data/annotations/match_001_annotations.json', 'w', encoding='utf-8') as f:
    json.dump(video_info, f, ensure_ascii=False, indent=2)
```

### 2. 标注步骤

#### 步骤1：观看视频

1. 完整观看视频，了解比赛节奏
2. 记录大致的回合数量
3. 注意特殊情况（如重发球、争议球等）

#### 步骤2：定位动作

**发球动作（round_start）**：

1. 找到发球方准备发球的画面
2. 逐帧播放，定位球拍击球瞬间
3. 记录击球瞬间的帧号（center_frame）
4. 向前推10-15帧作为start_frame
5. 向后推10-15帧作为end_frame

**球落地动作（round_end）**：

1. 找到球即将落地的画面
2. 逐帧播放，定位球接触地面瞬间
3. 记录接触瞬间的帧号（center_frame）
4. 向前推10-15帧作为start_frame
5. 向后推10-15帧作为end_frame

#### 步骤3：填写标注

```json
{
  "action_id": 1,
  "class": "round_start",
  "start_frame": 100,
  "end_frame": 120,
  "center_frame": 110,
  "description": "第1回合发球",
  "notes": "正手发球，高远球"
}
```

#### 步骤4：保存标注

将标注添加到annotations数组中：

```json
{
  "video_name": "match_001",
  "total_frames": 5400,
  "frame_rate": 30,
  "annotations": [
    {
      "action_id": 1,
      "class": "round_start",
      "start_frame": 100,
      "end_frame": 120,
      "description": "第1回合发球"
    },
    {
      "action_id": 2,
      "class": "round_end",
      "start_frame": 450,
      "end_frame": 470,
      "description": "第1回合球落地"
    }
  ]
}
```

### 3. 批量标注技巧

#### 3.1 使用快捷键

在VLC中：
- `空格`：播放/暂停
- `E`：下一帧
- `Shift+左箭头`：上一帧
- `Ctrl+T`：显示时间

#### 3.2 标注顺序

1. 先标注所有发球动作
2. 再标注所有球落地动作
3. 最后检查配对关系

#### 3.3 记录辅助信息

```json
{

  "class": "round_start",

  "end_frame": 120,
  "description": "第1回合发球",
  "notes": "正手发球，高远球",

  "difficulty": "easy"
```

## 标注规范

### 1. 帧范围选择

#### 1.1 发球动作

**标注范围**：击球前10帧 ~ 击球后10帧

```
击球前10帧 ← [击球瞬间] → 击球后10帧
    ↓           ↓            ↓
start_frame  center_frame  end_frame
```

**示例**：
- 击球瞬间：第110帧
- start_frame：100
- end_frame：120

#### 1.2 球落地动作

**标注范围**：落地前10帧 ~ 落地后10帧

```
落地前10帧 ← [落地瞬间] → 落地后10帧
    ↓           ↓            ↓
start_frame  center_frame  end_frame
```

**示例**：
- 落地瞬间：第460帧
- start_frame：450
- end_frame：470

### 2. 特殊情况处理

#### 2.1 重发球

如果发球失误需要重发：

```json
{
  "action_id": 1,
  "class": "round_start",
  "start_frame": 100,
  "end_frame": 120,
  "description": "第1回合发球（第1次）",
  "notes": "发球失误，重发"
},
{
  "action_id": 2,
  "class": "round_start",
  "start_frame": 150,
  "end_frame": 170,
  "description": "第1回合发球（第2次）",
  "notes": "重发成功"
}
```

#### 2.2 争议球

如果球落点有争议：

```json
{
  "action_id": 10,
  "class": "round_end",
  "start_frame": 1450,
  "end_frame": 1470,
  "description": "第5回合球落地",
  "notes": "争议球，裁判判定出界",
  "quality": "uncertain"
}
```

#### 2.3 画面遮挡

如果关键动作被遮挡：

```json
{
  "action_id": 15,
  "class": "round_start",
  "start_frame": 2100,
  "end_frame": 2120,
  "description": "第8回合发球",
  "notes": "部分遮挡，但可见击球动作",
  "quality": "fair"
}
```

### 3. 质量标准

#### 3.1 优质标注（Good）

- ✅ 动作清晰可见
- ✅ 帧范围准确
- ✅ 无遮挡或干扰
- ✅ 光照良好

#### 3.2 合格标注（Fair）

- ✅ 动作基本可见
- ✅ 帧范围基本准确
- ⚠️ 轻微遮挡或模糊
- ⚠️ 光照一般

#### 3.3 不合格标注（Poor）

- ❌ 动作不清晰
- ❌ 帧范围不准确
- ❌ 严重遮挡
- ❌ 光照很差

**建议**：不合格的标注应该删除或重新标注。

## 质量检查

### 1. 自检清单

标注完成后，请检查以下内容：

- [ ] 每个回合都有发球和球落地配对
- [ ] 帧范围合理（start_frame < end_frame）
- [ ] 动作类别正确（round_start / round_end）
- [ ] action_id唯一且连续
- [ ] JSON格式正确（无语法错误）
- [ ] 描述信息完整

### 2. 验证工具

#### 2.1 使用系统验证

```bash
cd 02_code
python data_preprocess.py
# 选择 [3] 验证标注文件
```

#### 2.2 手动验证脚本

```python
import json

def validate_annotation(annotation_path):
    with open(annotation_path, 'r', encoding='utf-8') as f:
        data = json.load(f)
    
    errors = []
    
    # 检查必要字段
    required_fields = ['video_name', 'total_frames', 'frame_rate', 'annotations']
    for field in required_fields:
        if field not in data:
            errors.append(f"缺少字段: {field}")
    
    # 检查标注
    for i, ann in enumerate(data['annotations']):
        # 检查类别
        if ann['class'] not in ['round_start', 'round_end']:
            errors.append(f"标注{i+1}: 类别无效")
        
        # 检查帧范围
        if ann['start_frame'] >= ann['end_frame']:
            errors.append(f"标注{i+1}: 帧范围无效")
    
    if errors:
        print("发现错误:")
        for error in errors:
            print(f"  - {error}")
        return False
    else:
        print("✓ 验证通过")
        return True

# 使用示例
validate_annotation('01_data/annotations/match_001_annotations.json')
```

### 3. 常见错误

#### 错误1：帧范围重叠

```json
// 错误示例
{
  "action_id": 1,
  "start_frame": 100,
  "end_frame": 150
},
{
  "action_id": 2,
  "start_frame": 120,  // 与上一个重叠
  "end_frame": 170
}
```

**解决方案**：确保不同动作的帧范围不重叠。

#### 错误2：类别错误

```json
// 错误示例
{
  "class": "serve"  // 应该是 "round_start"
}
```

**解决方案**：使用正确的类别标识。

#### 错误3：帧号超出范围

```json
// 错误示例（视频总帧数5400）
{
  "start_frame": 5390,
  "end_frame": 5410  // 超出范围
}
```

**解决方案**：确保帧号在有效范围内。
