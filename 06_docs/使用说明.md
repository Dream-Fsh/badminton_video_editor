## 一、环境准备与安装

### 1.1 系统要求

| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| 操作系统 | Windows 10 / Ubuntu 18.04 / macOS 10.15 | Windows 11 / Ubuntu 22.04 |
| Python | 3.8 | 3.10 |
| 内存 | 8 GB | 16 GB |
| 显卡 | 支持 CUDA 的 4GB 显存 | NVIDIA RTX 3060 12GB |
| 存储空间 | 10 GB 可用空间 | 50 GB 可用空间 |
| FFmpeg | 4.0+ | 5.0+ |

### 1.2 安装步骤

#### 步骤 1：克隆或解压项目

```bash
cd d:/Projects/python/badminton_video_editor
```

#### 步骤 2：创建虚拟环境（强烈推荐）

```bash
# Windows
python -m venv .venv
.venv\Scripts\activate

# Linux / macOS
python3 -m venv .venv
source .venv/bin/activate
```

#### 步骤 3：安装 Python 依赖

```bash
pip install -r requirements.txt
```

`requirements.txt` 内容：

```
Flask>=2.0.0
Werkzeug>=2.0.0
torch>=1.8.0
torchvision>=0.9.0
opencv-python>=4.5.0
numpy>=1.19.0
pyyaml>=5.4.0
tqdm>=4.60.0
```

> **注意**：如需 GPU 加速，请根据 CUDA 版本安装对应 PyTorch。访问 [pytorch.org](https://pytorch.org) 获取安装命令。

#### 步骤 4：安装 FFmpeg

**Windows**：
1. 下载：https://ffmpeg.org/download.html
2. 解压至 `C:\ffmpeg`
3. 将 `C:\ffmpeg\bin` 添加到系统环境变量 PATH
4. 验证：`ffmpeg -version`

**Ubuntu**：
```bash
sudo apt-get update
sudo apt-get install ffmpeg
```

**macOS**：
```bash
brew install ffmpeg
```

#### 步骤 5：下载预训练权重

1. 下载 `rgb_imagenet.pt`（I3D Kinetics + ImageNet 预训练权重）
2. 放置到 `03_model/pretrained/rgb_imagenet.pt`
3. （可选）下载微调后的羽毛球检测模型至 `runs/detect/03_model/shuttlecock_yolov8n/weights/best.pt`

## 三、Web 界面使用指南

### 3.1 用户注册与登录

1. 访问 `http://127.0.0.1:5000/register` 注册账号
2. 用户名至少 3 个字符，密码至少 6 个字符
3. 邮箱为可选项，用于找回密码接收验证码
4. 系统默认创建管理员账号，可通过数据库直接管理

### 3.2 视频上传

1. 登录后进入首页，拖拽或点击选择视频文件
2. 选择比赛类型：**单打** 或 **双打**
3. 支持格式：MP4、AVI、MOV、MKV、WMV
4. 单个文件最大 2GB
5. 上传成功后自动跳转至智能检测页面

### 3.3 智能检测

1. 点击**开始检测**按钮，系统自动在后台执行：
   - 加载最新训练好的模型（`03_model/trained/best_model_*.pth`）
   - I3D 滑动窗口推理识别发球与球落地
   - YOLO 辅助验证运动员位置与羽毛球状态
   - 回合配对与约束过滤
2. 前端实时显示进度百分比与当前处理阶段
3. 检测完成后自动跳转至回合播放器

### 3.4 回合播放与下载

1. **完整视频预览**：支持播放原始视频，拖动进度条定位
2. **回合列表**：左侧显示检测到的所有回合，点击即可跳转
3. **单回合下载**：点击下载按钮，系统使用 FFmpeg 精确剪辑该回合片段
4. 下载文件保存在 `04_output/` 目录，文件名格式：`round_{序号}_{时间戳}.mp4`

### 3.5 管理员后台

管理员账号登录后访问 `/admin`，可执行：
- **用户管理**：查看/禁用/删除用户、重置密码
- **操作日志**：审计用户登录、检测、下载等行为
- **邮件测试**：验证 SMTP 邮件服务连通性
- **数据统计**：查看注册用户量、视频处理量等

## 四、命令行使用指南

### 4.1 数据预处理（选项 1）

```
请选择操作：
  [1] 批量提取视频帧
  [2] 合并标注文件
  [3] 验证标注文件
  [4] 执行全部操作
```

**推荐**：选择 `[4]` 一键执行全部预处理。

预处理结果：
- 视频帧：`01_data/processed_frames/{视频名}/`
- 合并标注：`01_data/annotations/merged_labels.json`

### 4.2 模型训练（选项 2）

训练过程自动显示：
- 每 epoch 的训练损失与准确率
- 验证集损失与准确率
- 当前最优模型保存路径

训练完成后，模型保存至：
- `03_model/trained/best_model_YYYYMMDD_HHMMSS.pth` — 验证集最优模型
- `03_model/trained/final_model_YYYYMMDD_HHMMSS.pth` — 最终模型

**训练时间参考**：
- GPU（RTX 3060）：约 2–4 小时（30 epochs）
- CPU（i7-12700）：约 10–20 小时（30 epochs）

### 4.3 模型预测（选项 3）

1. 系统自动列出可用模型，默认选择最新的最优模型
2. 列出 `01_data/raw_videos/` 下的所有视频：
   - 输入视频编号：预测单个视频
   - 输入 `0`：批量预测所有视频
3. 预测结果保存至 `04_output/predictions/{视频名}_predictions.json`

预测结果 JSON 示例：
```json
{
  "video_name": "match_001",
  "prediction_time": "2026-05-01 14:30:00",
  "total_predictions": 48,
  "rounds": [
    {
      "round_id": 1,
      "start_time": 12.5,
      "end_time": 18.3,
      "duration": 5.8,
      "start_confidence": 0.92,
      "end_confidence": 0.88
    }
  ],
  "statistics": {
    "total_rounds": 24,
    "average_duration": 6.2
  }
}
```

### 4.4 视频剪辑（选项 4）

系统根据预测结果自动：
1. 读取预测 JSON 中的回合时间戳
2. 应用前后缓冲（`start_pre_padding=0.2s`, `end_post_padding=0.8s`）
3. FFmpeg 批量导出回合片段

剪辑结果保存至：
```
04_output/clips/{视频名}/
├── {视频名}_round_001.mp4
├── {视频名}_round_002.mp4
├── ...
└── {视频名}_clips_info.json
```

### 4.5 完整流程（选项 5）

一键执行 **预测 + 剪辑**，适用于已完成模型训练后处理新视频的场景。

## 五、数据准备规范

### 5.1 视频文件要求

| 属性 | 建议值 | 说明 |
|------|--------|------|
| 格式 | MP4 (H.264) | 兼容性最佳 |
| 分辨率 | ≥ 720p | 清晰展示运动员与羽毛球 |
| 帧率 | 25–30 FPS | 保证动作连续性 |
| 视角 | 固定高位俯拍 | 全场可见，减少遮挡 |
| 时长 | 完整比赛或单局 | 系统支持任意长度 |

### 5.2 视频存放位置

```
01_data/raw_videos/
├── match_001.mp4
├── match_002.mp4
└── ...
```

### 5.3 标注文件规范

如需自定义训练，需为每个视频创建 JSON 标注文件：

**文件名规则**：`{视频名}_annotations.json`

**示例** (`match_001_annotations.json`)：

```json
{
  "video_name": "match_001",
  "total_frames": 5400,
  "frame_rate": 30,
  "annotations": [
    {
      "action_id": 1,
      "class": "round_start",
      "start_frame": 100,
      "end_frame": 120,
      "description": "第 1 回合发球"
    },
    {
      "action_id": 2,
      "class": "round_end",
      "start_frame": 450,
      "end_frame": 470,
      "description": "第 1 回合球落地"
    }
  ]
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `video_name` | string | 是 | 视频名称（不含扩展名） |
| `total_frames` | int | 是 | 视频总帧数 |
| `frame_rate` | int | 是 | 帧率 |
| `action_id` | int | 是 | 动作唯一 ID |
| `class` | string | 是 | `round_start` 或 `round_end` |
| `start_frame` | int | 是 | 动作开始帧 |
| `end_frame` | int | 是 | 动作结束帧 |
| `description` | string | 否 | 动作描述 |

**标注技巧**：
- **发球（round_start）**：以球拍击球瞬间为中心，前后各取 10–15 帧
- **球落地（round_end）**：以球触地瞬间为中心，前后各取 10–15 帧
- 确保标注的帧范围在视频总帧数内

**获取视频信息**：

```python
import cv2
cap = cv2.VideoCapture('match_001.mp4')
print(f"总帧数: {int(cap.get(cv2.CAP_PROP_FRAME_COUNT))}")
print(f"帧率: {cap.get(cv2.CAP_PROP_FPS)}")
cap.release()
```

### 5.4 标注文件存放位置

```
01_data/annotations/
├── match_001_annotations.json
├── match_002_annotations.json
└── merged_labels.json    # 系统自动生成
```

## 六、配置文件说明

系统通过 `05_config/config.yaml` 统一管理所有参数。

### 6.1 常用参数速查

#### 训练参数

```yaml
training:
  batch_size: 1           # 显存 < 4GB 必须设为 1
  epochs: 30              # 小数据集可适当增加
  learning_rate: 0.001    # 小数据集可用较高值
  sequence_length: 16     # I3D 输入帧数（降低可减少显存占用）
```

#### 预测参数

```yaml
prediction:
  confidence_threshold: 0.1   # 阈值越低，召回率越高，误检可能增加
  sliding_window_stride: 16   # 步长越大速度越快（8→16 速度翻倍）
  athlete_detection:
    enabled: true             # 是否启用 YOLO 辅助检测
```

#### 剪辑参数

```yaml
video_editing:
  round_extraction:
    pre_padding: 2.0      # 回合开始前保留秒数
    post_padding: 2.0     # 回合结束后保留秒数
    min_round_duration: 2.0
    max_round_duration: 30.0
```

#### ROI 裁剪参数

```yaml
preprocessing:
  roi:
    enabled: true         # 是否启用 ROI 裁剪
    x_offset: 165         # 裁剪起点 X
    y_offset: 0           # 裁剪起点 Y
    width: 950            # 裁剪宽度
    height: 720           # 裁剪高度
```

### 6.2 调整参数示例

**场景 1：显存不足（CUDA out of memory）**

```yaml
training:
  batch_size: 1
  sequence_length: 16       # 从 32 降低至 16
prediction:
  batch_size: 1
```

**场景 2：预测结果过多（误检率高）**

```yaml
prediction:
  confidence_threshold: 0.3   # 从 0.1 提高
  athlete_detection:
    constraints:
      min_player_count: 2     # 要求检测到 2 名运动员
```

**场景 3：处理速度优先**

```yaml
prediction:
  sliding_window_stride: 16   # 增大步长
  athlete_detection:
    sample_rate: 5            # 降低 YOLO 检测频率
```

## 七、常见问题排查

### Q1：启动时提示 `Config file not found`

**原因**：工作目录不正确，导致配置文件查找失败。

**解决**：
- Web 模式：确保从项目根目录执行 `python run_web.py`
- CLI 模式：从项目根目录执行 `python main.py`，或直接进入 `02_code` 目录执行 `python main.py`

### Q2：`CUDA out of memory` 显存不足

**解决**：
1. 降低训练/预测的 `batch_size` 至 1
2. 降低 `sequence_length`（如从 32 降至 16）
3. 关闭其他占用显存的程序
4. 使用 CPU 模式运行（将 `system.device` 设为 `"cpu"`）

### Q3：`FFmpeg not found` 或剪辑失败

**解决**：
1. 命令行执行 `ffmpeg -version` 验证是否安装
2. 确认 FFmpeg 所在目录已加入系统 PATH
3. Windows 用户需重启命令行窗口使 PATH 生效
4. 重新下载 FFmpeg 并正确配置环境变量

### Q4：Web 界面检测任务卡住（进度不动）

**原因**：可能为检测线程异常退出，但状态未清理（僵尸状态）。

**解决**：
- 系统已内置僵尸线程自动检测，刷新页面后重新点击开始检测即可
- 如反复出现，检查 `03_model/trained/` 目录是否存在 `best_model_*.pth` 模型文件

### Q5：预测结果不准确（回合数不对）

**解决**：
1. 检查训练数据量：每类至少标注 30 个样本
2. 降低 `confidence_threshold`（如从 0.3 降至 0.1）提高召回率
3. 增加训练轮数（`epochs: 50`）
4. 检查标注质量，确保动作中心帧标注准确
5. 调整 YOLO 约束参数（绿色场地边界、运动员人数等）

### Q6：视频上传后无法播放

**解决**：
1. 检查视频格式是否在支持列表内（MP4/AVI/MOV/MKV/WMV）
2. 尝试用 FFmpeg 转码为标准 H.264 MP4：
   ```bash
   ffmpeg -i input.avi -c:v libx264 -crf 23 -preset fast -c:a aac output.mp4
   ```
3. 清除浏览器缓存后刷新页面

### Q7：邮件验证码发送失败

**解决**：
1. 系统默认使用**开发模式**，验证码直接显示在 Web 页面上
2. 如需启用真实邮件服务：
   - 编辑 `config/email_config.json`，填写 SMTP 服务器信息
   - 设置 `"enabled": true`
   - 或通过环境变量 `EMAIL_SENDER` 和 `EMAIL_PASSWORD` 配置

## 八、最佳实践

### 8.1 数据标注

- **样本数量**：每类（发球/球落地）至少 30 个标注样本，建议 50+ 个
- **样本质量**：确保动作峰值帧位于标注窗口中心，边界清晰
- **样本多样性**：覆盖不同光照、角度、场地颜色的视频
- **一致性**：同一批标注由同一人完成，减少主观偏差

### 8.2 模型训练

- **初次实验**：先用 10 epochs 快速验证数据流与模型收敛性
- **正式训练**：增加至 30 epochs，观察验证集准确率变化
- **过拟合判断**：训练集准确率 >> 验证集准确率（差距 >10%）时，增加数据增强或正则化
- **模型选择**：优先使用 `best_model_*`（验证集最优），而非 `final_model_*`

### 8.3 模型预测

- **速度与精度权衡**：
  - 高精度场景：`stride=8`, `confidence_threshold=0.1`
  - 高速度场景：`stride=16`, `confidence_threshold=0.3`
- **批量处理**：一次上传多个视频，利用异步检测提高效率
- **结果审查**：首次使用新视频源时，人工抽查 5–10 个回合，验证检测质量

### 8.4 视频剪辑

- **缓冲时间**：根据比赛节奏调整 `start_pre_padding` 和 `end_post_padding`
  - 快节奏比赛（双打）：0.2s / 0.5s
  - 慢节奏比赛（单打）：0.5s / 1.0s
- **画质与体积**：`crf=18` 高质量，`crf=23` 默认平衡，`crf=28` 小体积
- **存储规划**：原始 1GB 视频约产生 300–500MB 回合片段（取决于回合数量）

### 8.5 系统维护

- **定期备份**：`03_model/trained/` 下的模型权重、`01_data/annotations/` 下的标注文件
- **日志查看**：`04_output/logs/` 目录记录运行日志，出错时优先查看
- **数据库备份**：`data/users.db` 存储用户数据，定期复制备份
