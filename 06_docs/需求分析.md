# 羽毛球视频智能剪辑系统需求分析

## 3.1 功能性需求

### (1) 系统用例分析

#### 系统角色定义

1. **普通用户**：羽毛球运动员、教练、比赛分析人员等，需要上传比赛视频并获取剪辑回合
2. **管理员**：系统运维人员，负责用户管理、系统监控和后台配置

#### 用例描述

**普通用户用例：**

1. **用户注册**：新用户通过邮箱和密码创建账户，系统验证信息有效性后存储至数据库
2. **用户登录**：已注册用户凭账号密码进行身份验证，系统建立会话状态并授权访问
3. **视频上传**：用户选择本地羽毛球比赛视频文件（支持MP4、AVI等格式）并提交至服务器
4. **参数配置**：用户设置检测参数（如比赛类型单打/双打、ROI区域选择、置信度阈值）
5. **智能检测启动**：用户触发视频分析流程，系统异步执行I3D动作识别与YOLO目标检测
6. **进度查询**：用户在检测过程中实时查看处理进度和当前状态
7. **回合预览**：检测完成后，用户在Web界面按时间线浏览识别出的回合片段
8. **视频播放**：用户点击特定回合时，系统加载并播放该段视频内容
9. **片段下载**：用户选择感兴趣的一个或多个回合片段，系统调用FFmpeg剪辑并打包下载
10. **密码重置**：忘记密码时通过邮箱接收验证码并重设登录密码

**管理员用例：**

1. **用户管理**：管理员查看所有注册用户列表，可禁用/启用账户、重置用户密码
2. **操作日志审计**：管理员查看系统所有操作记录，包括登录、上传、检测等事件
3. **系统配置管理**：管理员修改全局配置参数（如最大文件大小、允许的视频格式）
4. **模型版本管理**：管理员上传或切换I3D、YOLO等深度学习模型文件
5. **资源监控**：管理员查看服务器CPU/内存使用率、磁盘空间和当前任务队列状态

#### 用例3：忘记密码

| 字段 | 内容 |
|------|------|
| 用例名称 | 忘记密码 |
| 用例编号 | UC-003 |
| 参与者 | 普通用户 |
| 前置条件 | 1. 用户已注册且记住注册邮箱<br>2. 系统邮件服务已配置并可用 |
| 后置条件 | 1. 用户密码被重置为新密码<br>2. 用户可使用新密码登录系统<br>3. 系统记录密码重置日志 |
| 约束规则 | 1. **验证码有效期**：6位数字验证码有效期为5分钟<br>2. **发送频率限制**：同一邮箱60秒内只能发送一次验证码<br>3. **重置次数限制**：同一邮箱24小时内最多重置密码3次<br>4. **密码历史检查**：新密码不能与最近5次使用的密码相同<br>5. **验证码长度**：必须是6位数字，有效期内只能使用一次 |
| 基本事件流 | 1. 用户在登录页面点击"忘记密码"链接<br>2. 系统显示忘记密码页面，包含邮箱输入框<br>3. 用户输入注册时使用的邮箱地址<br>4. 用户点击"发送验证码"按钮<br>5. 系统验证邮箱是否存在于数据库中<br>6. 系统生成6位数字验证码<br>7. 系统将验证码存储至数据库（设置5分钟有效期）<br>8. 系统通过SMTP邮件服务发送验证码至用户邮箱<br>9. 用户在忘记密码页面输入收到的验证码<br>10. 用户输入新密码和确认密码<br>11. 用户点击"重置密码"按钮<br>12. 系统验证验证码是否正确且未过期<br>13. 系统验证两次输入的新密码是否一致<br>14. 验证通过后，系统对新密码进行加盐哈希处理<br>15. 系统更新数据库中该用户的密码字段<br>16. 系统提示"密码重置成功"，跳转至登录页面<br>17. 用户使用新密码登录系统 |
| 备选事件流 | **A1：开发模式下显示验证码**<br>1. 系统处于开发模式（DEBUG=True）<br>2. 验证码发送后，系统在页面上直接显示验证码<br>3. 方便开发者测试，无需实际接收邮件 |
| 异常事件流 | **E1：邮箱不存在**<br>1. 系统未找到输入邮箱对应的用户<br>2. 系统提示"该邮箱未注册"<br>3. 用户检查邮箱地址或前往注册页面<br><br>**E2：邮件发送失败**<br>1. 系统SMTP服务配置错误或网络问题<br>2. 系统提示"邮件发送失败，请稍后重试"<br>3. 系统记录邮件发送失败日志<br><br>**E3：验证码过期**<br>1. 用户输入的验证码已超过5分钟有效期<br>2. 系统提示"验证码已过期，请重新获取"<br>3. 用户点击"重新发送"获取新验证码<br><br>**E4：验证码错误**<br>1. 用户输入的验证码与系统存储的不匹配<br>2. 系统提示"验证码错误"<br>3. 用户可重新输入或获取新验证码 |

|------|------|

| 用例编号 | UC-004 |
| 参与者 | 普通用户 |

| 后置条件 | 1. 视频文件被保存至服务器指定目录<br>2. 视频信息（文件名、大小、时长）被记录到数据库<br>3. 用户可对上传的视频执行后续操作 |
| 约束规则 | 1. **文件大小限制**：单个视频文件最大2GB<br>2. **支持格式**：仅支持MP4、AVI、MOV、MKV格式<br>3. **并发上传限制**：单个用户只能同时上传1个文件<br>4. **文件名规则**：系统自动重命名为UUID格式，避免文件名冲突<br>5. **视频时长限制**：上传视频时长不超过3小时<br>6. **存储空间检查**：上传前检查服务器剩余空间至少需大于文件大小 |

| 备选事件流 | **A1：拖拽上传**<br>1. 用户直接将视频文件拖拽到上传区域<br>2. 系统自动验证并开始上传流程<br><br>**A2：批量上传**<br>1. 用户选择多个视频文件<br>2. 系统创建队列依次上传，显示总体进度<br>3. 所有文件上传完成后统一跳转至检测页面 |
| 异常事件流 | **E1：文件格式不支持**<br>1. 用户选择的文件格式不在允许列表中<br>2. 系统提示错误并列出支持格式<br>3. 用户需重新选择文件<br><br>**E2：文件大小超限**<br>1. 视频文件超过2GB限制<br>2. 系统拒绝上传并提示限制大小<br>3. 建议用户压缩视频或分割处理<br><br>**E3：网络中断**<br>1. 上传过程中网络连接断开<br>2. 系统检测到传输失败<br>3. 保留已上传的部分，用户可断点续传（如浏览器支持） |

#### 用例5：智能检测启动

| 字段 | 内容 |
|------|------|
| 用例名称 | 智能检测启动 |
| 用例编号 | UC-005 |
| 参与者 | 普通用户 |
| 前置条件 | 1. 用户已成功登录系统<br>2. 用户已上传有效的羽毛球比赛视频文件<br>3. 系统检测服务处于可用状态 |
| 后置条件 | 1. 系统创建后台检测任务并开始处理<br>2. 用户可查询任务进度和最终结果 |
| 约束规则 | 1. **任务队列限制**：系统最多同时处理3个视频检测任务<br>2. **视频时长限制**：检测视频时长不能超过3小时<br>3. **GPU内存要求**：使用GPU模式时至少需4GB显存<br>4. **进度更新频率**：检测进度每5%更新一次数据库<br>5. **超时处理**：单个视频检测超过2小时自动终止<br>6. **模型路径限制**：I3D和YOLO模型文件路径必须在系统配置目录内 |
| 基本事件流 | 1. 用户在视频上传页面点击"开始检测"按钮<br>2. 系统验证视频文件格式和大小符合要求<br>3. 系统初始化I3D动作识别模型和YOLOv8目标检测模型<br>4. 系统创建异步处理线程，将视频拆解为图像帧序列<br>5. 系统采用滑动窗口策略对帧序列进行I3D推理，识别发球和球落地动作<br>6. 系统并行执行YOLO检测，追踪运动员位置和羽毛球运动状态<br>7. 系统融合双重检测结果，生成回合分割时间点<br>8. 系统将处理进度实时更新至数据库<br>9. 处理完成后，系统生成JSON格式的检测报告和预览缩略图<br>10. 系统通知用户检测完成 |
| 备选事件流 | **A1：用户选择自定义参数**<br>1. 用户在检测前设置ROI区域、置信度阈值等参数<br>2. 系统使用用户参数而非默认配置执行检测<br><br>**A2：检测过程中用户取消**<br>1. 用户点击"取消检测"按钮<br>2. 系统停止当前处理线程并清理临时文件<br>3. 系统更新任务状态为"已取消" |
| 异常事件流 | **E1：视频格式不支持**<br>1. 系统检测到视频格式不在允许列表中<br>2. 系统提示错误信息并拒绝启动检测<br><br>**E2：模型加载失败**<br>1. 系统无法加载预训练的I3D或YOLO模型权重<br>2. 系统记录错误日志并通知管理员<br>3. 系统向用户显示"服务暂时不可用"提示<br><br>**E3：GPU显存不足**<br>1. 检测过程中GPU显存溢出<br>2. 系统自动回退至CPU模式继续执行<br>3. 系统记录性能警告日志 |

#### 用例6：回合预览与播放

| 字段 | 内容 |
|------|------|
| 用例名称 | 回合预览与播放 |
| 用例编号 | UC-006 |
| 参与者 | 普通用户 |
| 前置条件 | 1. 用户已成功登录系统<br>2. 用户的视频文件已完成智能检测处理<br>3. 系统已生成回合分割时间点和预览数据 |
| 后置条件 | 1. 用户获得回合的时间线视图<br>2. 用户可选择特定回合进行播放或下载 |

## 3.2 非功能性需求

### 系统可用性

系统应提供直观、易用的用户界面，确保羽毛球教练、运动员等非技术用户能够快速上手。Web界面采用响应式设计，适配桌面端和移动端浏览器，主要功能操作在3次点击内完成。界面布局符合用户认知习惯，视频上传、检测启动、结果查看等核心流程提供明确引导和状态反馈。系统需提供详细的操作帮助文档和错误提示信息，当用户操作失误时给予清晰的纠正指导。关键功能如视频播放器应支持标准快捷键操作，减少用户学习成本。系统平均培训时间不超过30分钟，新用户能够在无专业指导情况下独立完成视频剪辑任务。

### 系统可靠性

系统应保证7×24小时稳定运行，核心服务可用性不低于99.5%。视频检测过程中如遇意外中断（如服务器重启、网络故障），系统应支持任务恢复机制，从断点继续执行而非重新开始。所有用户数据（上传视频、检测结果、账户信息）需定期备份，确保数据丢失风险低于0.1%。系统应具备异常检测和自愈能力，当深度学习模型加载失败或GPU内存溢出时，自动降级为CPU模式或切换到备用模型。关键操作如用户认证、视频处理需实现事务性保证，避免出现数据不一致状态。系统平均无故障时间（MTBF）应大于720小时，平均修复时间（MTTR）不超过2小时。

### 可支持性

系统应提供完整的运维支持方案。采用模块化架构设计，各功能模块之间耦合度低，便于单独升级或替换。系统配置全部通过YAML文件管理，无需修改代码即可调整检测参数、文件路径、服务端口等设置。提供详细的API接口文档和开发指南，支持第三方系统集成。日志系统记录详细运行信息，包括用户操作、模型推理耗时、资源使用情况等，支持按时间、用户、操作类型筛选查询。管理员后台提供系统健康状态监控面板，实时显示CPU、内存、磁盘、GPU使用率。系统支持在线更新机制，可远程部署新版本模型权重和功能模块，减少停机维护时间。

### 安全性

系统需实现多层次安全防护机制。用户认证采用密码加盐哈希存储，传输过程使用Flask Session加密，防止密码明文泄露。用户上传的视频文件需进行格式和内容安全检查，防止恶意文件上传攻击。实施基于角色的访问控制（RBAC），普通用户只能访问自己的视频数据和检测结果，管理员需额外授权才能操作用户管理功能。所有用户操作需记录审计日志，支持溯源查询。系统应对SQL注入、跨站脚本（XSS）、跨站请求伪造（CSRF）等常见Web攻击具有防护能力。视频处理过程中产生的临时文件需及时清理，避免敏感信息残留。系统与外部服务（如SMTP邮件服务）的通信需使用加密通道，确保数据传输安全。定期进行安全漏洞扫描和渗透测试，及时修复发现的安全隐患。
